A new way to look at food

Paste a label. Get a rough score and a list of flagged additives.

It’s one static site: HTML, CSS, and plain JS. I wanted something I could run from GitHub Pages without a build step, and still hit a real API (OpenFoodFacts) for product photos and nutrition when the lookup works.

Repo on GitHub Open the embedded app

What problem it’s solving

Ingredient lists are long and easy to skim past. This is mostly a UI + data exercise: match additives from a small dictionary, fake a bit of “processing” with list length, then color-code a 0–100 score so you get something readable at a glance. Brand search fills in calories and a pack image when the API returns a hit.

Scan side

Input is comma-separated ingredients. The script checks against an additive list, uses list length as a stand-in for “how processed this probably is,” and maps the final number to green / yellow / red (80+, 50–79, under 50).

Brand side

There’s a local JSON-ish fallback for brands so the autocomplete isn’t empty offline. Online, it calls OpenFoodFacts search and shows macros, a few auto-written pros/cons from thresholds, and a product image when one exists.

Behavior (short list)

Nothing fancy under the hood — I was aiming for “works in a lab browser” and “easy to grade.”

  1. Scan animation — line sweeps the textarea, short delay, then results land in one column.
  2. Brand field — suggestions while typing; arrows, Enter, Escape behave like you’d expect.
  3. Login screen — email/password, create account, guest, logout. Accounts are stubbed in localStorage for the assignment.
  4. Art — a few inline SVGs for the hero and categories; live mode swaps in a photo from OFF when it can.

How it’s built

No React, no Vite, no npm script you have to remember. That was intentional so the folder is the whole project and you can open index.html directly (CORS aside for some fetches).

Stack (honest version)

· HTML + CSS (variables, clamp for type) · JavaScript in one page flow — no bundler · fetch() to OpenFoodFacts JSON search · localStorage for the demo “accounts” · SVGs drawn / inlined by hand (and a couple generated blocks in this page)

OpenFoodFacts is volunteer-maintained. The scores here are heuristics for a class demo, not nutrition advice.
