Grislis · Nova App

Food labels, decoded fast — with science-style scoring.

A browser-based tool I built to cut through marketing copy on packaging: scan ingredient text, surface additives, compare brand-level risk, and pull live nutrition and product photos when available.

View repository on GitHub Jump to embedded app

Why this project

Grocery aisles are full of long ingredient lists and vague claims. Nova is my answer: one place to paste a label, get additive flags and a clear health score, search brands with autocomplete, and optionally enrich results with real product data — without installing an app store build.

Label intelligence

Users paste comma-separated ingredients. The engine matches a curated additive knowledge base, applies a length-based ultra-processed proxy, and outputs semantic color bands: green (80–100), yellow (50–79), red (0–49).

Brand + product layer

Local profiles cover mainstream and niche CPG brands for instant fallback. Live search hits the OpenFoodFacts JSON search API to show calories, protein, package imagery, and auto-generated positives and negatives from nutrient thresholds.

What you can do in Nova

Everything runs as static HTML, CSS, and JavaScript — easy to host on GitHub Pages and demo on any machine with a browser.

High-speed scan animation

A scanning line and short timed “analysis” phase makes the label check feel responsive and intentional, then results populate in one panel.

Brand autocomplete

Suggestions appear as you type; keyboard navigation supports arrow keys, Enter, and Escape for a proper lookup experience.

Auth + guest entry

Start screen supports login, create-account (stored in browser localStorage for this demo), guest access, and logout — framing the tool like a real product shell.

Visual storytelling

Custom SVG art for hero and brand categories; live mode swaps in real pack photography from OpenFoodFacts when a product match exists.

Technologies & data sources

No React, no bundler — deliberate vanilla JS so the project stays portable and classroom-demo friendly. External data is fetched with the standard Fetch API.

Libraries & APIs (at a glance)

Vanilla JavaScript (ES modules not required) Semantic HTML5 CSS variables + clamp() for zoom-friendly type Fetch API → OpenFoodFacts search localStorage (session + demo accounts) Inline SVG assets

OpenFoodFacts is a community-driven open database; Nova queries it for product matches, then derives a simple nutrition score and bullet positives/negatives — educational heuristics, not medical advice.

Interface snapshots (illustrated)

Stylistic mockups of the real UI — useful when projecting at high zoom (Cmd +). The live app matches this structure: auth gate, dual-column workspace, results with score ring and lists.
